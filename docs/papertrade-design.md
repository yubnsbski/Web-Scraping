# ペーパートレード学習ループ 設計書（v1）

作成: 2026-07-10（オーケストレーター設計。architect/Codex がレート制限のため、オーナー承認済みの自走権限に基づきオーケストレーターが設計を担当）

## 目的

AIの売買判断が反復（サイクル）を通じて**改善するかを検証する**。実注文は一切行わない（AGENTS.md準拠、Phase 2 の合意済みステップ）。ループ:

```
AIが仮想売買を決定 → 現実準拠のシミュレーションで約定・評価
→ 実績を「学習材料（実績メモ）」化してAIに読ませる → 次サイクルの判断に反映 → 繰り返し
→ サイクル横断で改善したかを統計的に確認（運は排除）
```

## オーナー要件（2026-07-10）

1. 実注文なし。仮想売買のみ。
2. 売買方法は実際の日本株取引と同じにする（要調査 → 本書「市場メカニクス」）。
3. **銘柄別の重みづけ禁止**。許されるのは戦略スタイル（利回り重視/運用重視）と**セクター特性**（電力・通信=ディフェンシブ等）のみ。
4. 変化量は常にウォッチ可能に。管理者（オーナー)がチェックできる体制。
5. 完了まで自走、push まで自動承認。

## ループ設計（ウォークフォワード）

- 1サイクル = 5営業日（週次）。122営業日 − ウォームアップ60日 ≒ **12サイクル**（サイクル長は設定可能。3営業日にすれば約20サイクル）。
- 各サイクル: ①判断（サイクル初日の前日引けまでのデータのみ使用 — 先読み禁止をエンジンで強制）→ ②翌営業日寄付で約定 → ③期間中は保有 → ④サイクル末に評価 → ⑤実績メモ生成・保存 → 次サイクルの判断材料に。

### 判断レイヤー（決定論的コア + 限定的LLM）

- **コアは決定論的**（Gemini無料枠の厳守・再現性・オフラインテストのため）: 戦略プリセットがスコアリングで銘柄を選定。
  - `yield`（利回り重視): 配当利回り = dividend_per_share ÷ 直近終値 の上位から選定。
  - `momentum`（運用重視): 20日リターン上位 + ボラティリティフィルタ。
- **銘柄別の重みは存在しない**: 選定N銘柄への均等配分（equity÷N を単元に丸め）。設定にも銘柄別ウェイト項目を**作らない**。
- **セクターは考慮する**: JPX33業種をディフェンシブ（電力・ガス、情報・通信、食料品、医薬品、陸運）/シクリカル等にマップ。セクター上限（既定30%）とディフェンシブ比率の下限/上限を政策パラメータ化。
- **LLMの役割 = 政策調整（学習の主体)**: 各サイクル、過去の実績メモ（直近K件）をプロンプトに含め、**ホワイトリスト化された政策パラメータ**の調整案を厳格JSONで返させる:
  - `n_positions` 8–15 / `sector_cap` 0.20–0.40 / `defensive_ratio_target` 0.0–0.6 / `momentum_lookback` 10–40日 / `cash_buffer` 0.0–0.2 / `rebalance_threshold` など。
  - JSONスキーマ検証 + 範囲クランプ。失敗・予算超過・オフライン時は**前回パラメータ維持**（決定論フォールバック）。
  - 実LLM呼び出しは**1サイクルあたり最大1回**（12サイクルで12回 ≪ 日次予算1,500）。`llm/service.py` 経由必須。
- **実績メモ（学習材料)**: サイクル終了時にメトリクスから決定論的テンプレートで日本語メモを生成（何を選び、何が当たり/外れ、セクター寄与、政策変更の効果）。`local_docs/papertrade/run_<id>/memo_cycle_NN.md` に保存し、papertrade.sqlite にも記録。RAGインデックス投入はオプションフラグ（チャットAIが結果を語れるように）。

### 改善検証（学習 vs 運の分離）

同一データ・同一初期条件で3系統を必ず並走:

| Run | 内容 |
|---|---|
| A: adaptive | LLMフィードバックループ ON |
| B: control | 同一戦略・初期パラメータ**凍結**（学習なし） |
| C: benchmark | ①サイクル0ポートフォリオの buy&hold ②ユニバース均等ウェイト指数 |

指標（サイクル毎に永続化): 期間リターン、累積リターン、対ベンチマーク超過、勝率（勝ちポジション比率）、シャープレシオ（日次→年率換算）、最大ドローダウン、回転率、セクター配分。

改善判定: (1) Aの前半サイクル vs 後半サイクルの超過リターン平均・勝率の比較、(2) A−B のサイクル毎ペア差の平均と符号検定的な集計。**n が小さいため「参考値」であることをレポートに明記**（断定しない）。

### 管理者モニタリング（要件4）

- `data/runtime/papertrade.sqlite` に全履歴（runs / cycles / orders / fills / positions_eod / policy_history / memos / cycle_metrics）。
- 各サイクル終了ごとに `local_docs/papertrade/run_<id>/report.md` を追記更新（A/B/C の資産曲線、サイクル毎デルタ表、政策パラメータの変更履歴と理由）。最終レポートは自己完結HTML（`report.html`、依存なし）も生成。
- CLI: `investment-assistant papertrade run --strategy yield --cycles 12` / `papertrade report --run <id>` / `papertrade status`。
- 免責文をすべてのレポート・メモに自動付与。断定的推奨表現は生成しない。

## モジュール構成（新規・独立パッケージ）

```
src/investment_assistant/papertrade/
  __init__.py
  calendar.py    # daily_bars から営業日列を構築
  universe.py    # ユニバース選定 + 33業種セクターマップ（utf-8-sig注意）
  mechanics.py   # 呼値丸め・値幅制限クランプ・単元丸め・スリッページ・手数料・税
  account.py     # 現金・ポジション・注文・約定・実現損益（T+2記録）
  engine.py      # ウォークフォワード実行器（先読み禁止を型/実行時で強制）
  strategy.py    # yield / momentum プリセット + 均等配分 + セクター制約
  policy.py      # 政策パラメータ定義・範囲検証・クランプ
  advisor.py     # LLM政策調整（llm/service経由、オフラインは決定論フェイク）
  memo.py        # 実績メモ生成（日本語テンプレート）
  metrics.py     # サイクル指標 + 改善判定統計
  store.py       # papertrade.sqlite スキーマ・永続化
  runner.py      # A/B/C 実験のオーケストレーション + report.md/html
```

`cli.py` にサブコマンド追加のみ（webapi には触れない — ルートテーブルが並行セッション所有のため）。

## スプリント分割

- **P1（基盤）**: calendar / universe / mechanics / account / store + 単体テスト（単元丸め・呼値丸め・値幅クランプ・損益通算課税のゴールデンテスト）。
- **P2（エンジン+戦略）**: engine / strategy / policy / metrics / runner（B・C系統まで）+ CLI + report.md。オフラインで12サイクル完走するE2Eテスト。
- **P3（学習ループ）**: advisor（LLM政策調整、オフラインフェイク既定）/ memo / A系統 / 改善判定統計 / report.html / RAGインデックス投入オプション。

各スプリントで品質ゲート（pytest/ruff/mypy）+ qa-reviewer（金融計算のため必須）→ コミット。全完了で push。

## リスク・既知の制約

- 日足始値約定 + 固定スリッページは実際の板より粗い（v2: VWAP・出来高制約）。
- 配当キャッシュフロー除外（権利落ち日データなし）→ yield戦略のリターンが過小評価される点をレポートに明記。
- 履歴が約6ヶ月しかなく、サイクル数が少ない（12前後）→ 統計的検出力は限定的。データ拡充（並行セッションのバックフィル）後に再実験。
- LLM調整が「たまたま当たる」リスク → B(control) との比較を必須とし、単独の見栄えの良い数字で判断しない。

## コンプライアンス

実注文コード経路なし。信用・空売りなし。レポート・メモに免責文自動付与。断定的売買推奨の表現を生成しない。AGENTS.md の自動売買禁止は維持（本機能はシミュレーションのみ）。
