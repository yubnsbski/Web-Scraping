# Codex運用ワークフロー

## 自動化する作業

Codexは以下を自動で進める。

- 実装
- テスト追加と修正
- リファクタリング
- ドキュメント更新
- CI設定更新

## 承認ポイント

通常運用では、以下の前にユーザー承認を求める。

1. `git commit`
2. `git push`
3. PR作成
4. 本番反映
5. 秘密情報や本番設定に関わる変更

ユーザーが特定の作業で「commit、pushまでは自動承認」と明示した場合は、その作業範囲ではcommitとpushを自動で進めてよい。
ただし、本番反映、秘密情報、本番設定、APIキー、個人情報、実注文、自動売買に関わる変更は別途確認する。


## AI伴奏型ターミナル運用

ユーザーが「ターミナルにコピペできる様に」と指定した作業では、Codexは以下を守る。

- セットアップ、実行、確認、テスト、commit/pushのコマンドをbashブロックで提示する。
- `git remote add origin` は、`origin` が未設定の場合だけ実行する形にする。
- コマンドはホームディレクトリからでも `Web-Scraping/pyproject.toml` を探してリポジトリルートへ移動できる形にし、必要な変数だけ先頭にまとめる。
- Web取得では `fetch-url --dry-run` を先に提示し、許可された場合だけ `--save-text` とRAG連携へ進める。
- コマンド集は `docs/terminal_runbook.md` を優先して更新する。

## 品質ゲート

commit前に以下を実行する。

```bash
ruff check .
pytest
mypy src
```

## 禁止事項

- `.env` やAPIキーのコミット。
- Gemini APIの直接呼び出し。
- 単体テストでの実API呼び出し。
- 自動売買や断定的な売買推奨。
