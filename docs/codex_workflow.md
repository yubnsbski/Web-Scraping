# Codex運用ワークフロー

## 自動化する作業

Codexは以下を自動で進める。

- 実装
- テスト追加と修正
- リファクタリング
- ドキュメント更新
- CI設定更新

## 承認ポイント

通常運用では、以下の前にユーザー承認を求める。

1. `git commit`
2. PR作成
3. 本番反映
4. 秘密情報や本番設定に関わる変更

## 品質ゲート

commit前に以下を実行する。

```bash
ruff check .
pytest
mypy src
```

## 禁止事項

- `.env` やAPIキーのコミット。
- Gemini APIの直接呼び出し。
- 単体テストでの実API呼び出し。
- 自動売買や断定的な売買推奨。
